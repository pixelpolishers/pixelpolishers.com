Contributors Guide

To contribute to the Resolver project one needs to fork the repository. This repository is located at Github.

Fixing Bugs

Did you find a bug? Please submit a pull request. To submit a pull request we follow a strict guideline.

If you do not have a fork yet, fork the repository and add a remote to the repository:

git clone git@github.com:<Your Username>/resolver.git cd resolver git remote add upstream git@github.com:pixelpolishers/resolver.git

Next pull the latest changes:

git checkout master git pull upstream master git push origin

Now create a branch, you will be doing your changes in there:

git checkout -b hotfix/<issue id>

Adding Features

Adding new features is much appreciated. Please submit a pull request in order for your feature request to be evaluated.
This works pretty much the same as fixing bugs. First of all you need the repository:

git clone git@github.com:<Your Username>/resolver.git cd resolver git remote add upstream git@github.com:pixelpolishers/resolver.git

Next pull the latest changes. All new features should be based on the latest development branch which is called "develop".

git checkout develop git pull upstream develop git push origin

Now create a branch, you will be doing your changes in there:

git checkout -b feature/<your feature>
