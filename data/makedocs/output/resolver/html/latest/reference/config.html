Configuration Reference

Introduction

All resolver projects should contain a configuration file. Resolver uses JSON to configure a project, therefor a resolver.json is required within the root of your project.

Types

Some members within the configuration file are of a specific type. Resolver contains the following types.

Name Description
Configuration An object containing members describing a project's configuration.
Project An object containing members describing a project.

Root Members

The root of the configuration file can contain the following members.

Name Type Description Required
name string The name of the project. This should be in the form of vendor/name Yes
projects array The list with projects. This should be an array with objects of the type Project No
vendor string The path of the directory to store the dependencies in. The default value is vendor No
